SSL/TLS Best Practices

In 2018 we started this work because we wanted to understand how HTTPS was used for Municipal Websites in Portugal.

The main goal is offer a complete analysis of the atual state of the websites and make it possible for anyone to consult to see the current state.

The idea is to improve the current state because there are many problems.

Private Key and Certificate

A strong private key is needed to prevent attackers from carrying out impersonation attacks and it is necessary to have a valid and strong certificate, which grants the private key the right to represent a particular hostname

Configuration

  • Use Complete Certificate Chains
  • Use Secure Protocols
    1. SSL v2 is insecure and must not be used
    2. SSL v3 is insecure when used with HTTP and obsolete.
    3. TLS v1.0 is also a legacy protocol that shouldn't be used, but it's typically still necessary in practice
    4. TLS v1.1 and v1.2 are both without known security issues
  • Use Secure Cipher Suites
  • Select Best Cipher Suites
  • Use Forward Secrecy
  • Use Strong Key Exchange
  • Mitigate Known Problems
